一、Flutter 层（Dart）优化 (已完成)
- [x] 直接生成完整 YAML 配置字符串，不写文件
- [x] 通过 MethodChannel 异步传给 iOS 原生
- [x] 不做任何解析、不缓存配置
- [x] 禁止连续重复发送配置，加简单防抖（300ms）
- [x] 只发送配置内容，不传递指针、不跨进程调用 C/Go
- [x] UI 状态通过简单字段从共享内存读取，不阻塞
- [x] 不直接与 Network Extension 通信

二、iOS 原生主 App 层（Swift）优化 (已完成)
- [x] 接收 Flutter 传来的配置字符串
- [x] 仅写入 App Group 共享内存，不写文件
- [x] 使用简单状态位：0 = 空 1 = 已有配置
- [x] 共享内存读写放在后台串行队列，避免并发混乱
- [x] 不解析、不修改配置内容，完全透传
- [x] 不用 sendProviderMessage 传输大配置
- [x] 拓展启动前确保配置已写入内存
- [x] 配置读取后可选择清空，不留冗余数据
- [x] 不引用 Flutter Engine、不操作 UIKit 在非主线程

三、PacketTunnel Extension 优化（最核心） (已完成)
- [x] startTunnel 中立刻回调 completionHandler (nil)，绝不阻塞主线程
- [x] 所有 mihomo 启动 / 停止操作，放入后台串行队列执行
- [x] 配置只从 App Group 共享内存读取，不读磁盘
- [x] 只做基础判空校验，不做格式校验（避免崩溃）
- [x] setUp() 阶段调用 MihomoWarmup() 预初始化 Go 运行时
- [x] 使用内存启动接口：MobileStartWithMemory(configCStr)
- [x] 不使用文件路径接口
- [x] 不使用 UIKit、不加载 Flutter、不做任何磁盘 IO
- [x] 简单捕获异常，防止 Go panic 直接炸掉拓展
- [x] 编译必开：Require Only App-Extension-Safe API = YES, Other C Flags: -fapplication-extension
- [x] 配置字符串直接转为 C 指针传递，尽量减少内存拷贝
