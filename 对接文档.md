# Android Socket Protect/Mark 对接文档

## 目标

在无 root 的 `VpnService + tun(fd)` 模式下，将 core 发起的外连 socket 交给 Android 侧执行 `protect(fd)`，避免回环；可选执行 mark 回调。

## Go 导出接口

已在 mobile 导出层提供：

- `SetSocketProtector(protector SocketProtector)`
- `ClearSocketProtector()`

`SocketProtector` 接口：

- `ProtectSocket(fd int64, network string, address string) bool`
- `MarkSocket(fd int64, network string, address string) bool`

行为约定：

- `ProtectSocket` 返回 `false` 会导致该连接创建失败。
- `MarkSocket` 为可选能力，返回值不会阻断连接。

## Android 侧实现方式

gomobile 生成后，实现 `mobile.SocketProtector` 并注册到 core。

```kotlin
class CoreSocketProtector(
    private val vpnService: VpnService
) : mobile.SocketProtector {
    override fun protectSocket(fd: Long, network: String?, address: String?): Boolean {
        return vpnService.protect(fd.toInt())
    }

    override fun markSocket(fd: Long, network: String?, address: String?): Boolean {
        return true
    }
}
```

注册与清理：

```kotlin
val protector = CoreSocketProtector(vpnService)
mobile.Mobile.setSocketProtector(protector)

mobile.Mobile.clearSocketProtector()
```

## 建议接入时序

1. 建立 `VpnService` 并获得 tun fd。
2. 调用 `setSocketProtector(...)`。
3. 调用 core `start(...)`。
4. 停止 VPN 或退出前，调用 `clearSocketProtector()`。

## 推荐配置（无 root）

```yaml
tun:
  enable: true
  stack: system
  file-descriptor: 你的fd
  auto-route: false
  auto-redirect: false
  dns-hijack:
    - 0.0.0.0:53
    - "[::]:53"
```

## 兼容性与注意事项

- `ProtectSocket` 回调是高频路径，需尽量轻量，避免阻塞。
- `mark` 在无 root 场景通常不可依赖，建议保持 `markSocket` 直接返回 `true`。
- 若未注册 protector 或 protector 被清理，core 将不再触发 Android protect 回调。
