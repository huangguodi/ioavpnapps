# 更改密钥与服务器 API 接口地址指南

为了提升应用的安全性，防止核心加密密钥和服务器 API 地址被轻易逆向提取，本项目已将这些敏感信息从 Dart 源码迁移至底层 Native C++ 代码中，并在 C++ 层使用了 **XOR（异或）加密字节数组** 进行混淆。

当您需要更改服务器地址或更换加密密钥时，请修改对应的 C++ 源文件，然后重新编译应用。

## 1. 密钥类型与文档对齐说明

根据 `App V2 接口文档`，当前系统使用的加解密参数必须严格满足以下条件：
*   **AES Key (`getAesKey`)**: 用于请求体加密和响应体解密的 AES-256-GCM 密钥，**长度必须严格为 64 个十六进制字符** (32字节)。
*   **Obfuscate Key (`getObfuscateKey`)**: 用于对加密后数据进行二次 XOR 混淆的字符串密钥，**长度必须严格为 66 个字符**（目前兼容84字符）。
*   **Server URL Key (`getServerUrlKey`)**: 用于在 Dart 层解密（XOR）真实服务器 URL 字节数组的密钥。如果更改了服务器地址，需要重新生成字节数组并更新此密钥。

## 2. XOR 字节数组混淆说明 (Anti-Strings)

为了防止攻击者直接使用 `strings` 命令扫描 `.so` 或 `.dll` 文件提取出连续的完整密钥，C++ 代码中不再直接存储明文字符串，而是存储了经过异或加密（XOR Key = `0x5A`）的字节数组。

**如何生成新的加密字节数组？**

你可以使用以下 Python 脚本生成新的 C++ 字节数组：

```python
def encrypt_for_cpp(s, xor_key=0x5A):
    # 将字符串转为异或后的 hex 数组形式
    return ', '.join([hex(ord(c) ^ xor_key) for c in s])

# 在这里填入你的新明文密钥
new_aes = "这里填入64位新的AESKey..."
new_obf = "这里填入新的混淆Key..."
new_url = "这里填入新的URLKey..."

print("AES Array:\n", encrypt_for_cpp(new_aes))
print("OBF Array:\n", encrypt_for_cpp(new_obf))
print("URL Array:\n", encrypt_for_cpp(new_url))
```

将输出的十六进制数组（如 `0x62, 0x63, ...`）替换到 C++ 代码中的 `enc[]` 数组内。

## 3. Windows 端修改

Windows 端的密钥硬编码在 Runner 的入口文件中。

*   **文件路径**: `windows/runner/flutter_window.cpp`
*   **修改位置**: 搜索 `getAesKey`、`getObfuscateKey` 或 `getServerUrlKey`。
*   **示例**:
    ```cpp
    } else if (call.method_name() == "getAesKey") {
      const unsigned char enc[] = {
          0x62, 0x63, 0x1b, ... // 替换为你新生成的十六进制数组
      };
      result->Success(flutter::EncodableValue(decryptKey(enc, sizeof(enc), 0x5A)));
    }
    ```

## 4. Android 端修改

Android 端使用了 NDK 将密钥编译为 `.so` 动态链接库。

*   **文件路径**: `android/app/src/main/cpp/keys.cpp`
*   **修改位置**: 直接修改对应 JNI 函数中的 `enc[]` 数组。
*   **示例**:
    ```cpp
    JNIEXPORT jstring JNICALL
    Java_com_accelerator_tg_MainActivity_getAesKey(JNIEnv *env, jobject thiz) {
        const unsigned char enc[] = {
            0x62, 0x63, 0x1b, ... // 替换为你新生成的十六进制数组
        };
        std::string key = decryptKey(enc, sizeof(enc), 0x5A);
        return env->NewStringUTF(key.c_str());
    }
    ```

## 5. 更改服务器基础地址 (Base URL)

由于 URL 本身也是混淆的，如果你更改了服务器 IP 或域名：
1.  写一个简单的异或脚本（使用新的 `Server URL Key` 对新的 URL 字符串进行按位异或）。
2.  将生成的**十进制字节数组**更新到 `lib/services/api_service.dart` 中的 `_serverUrlBytes` 变量（注意：不要修改带有错别字的蜜罐变量 `_serverUlrBytes`）。
3.  将新的 `Server URL Key`（异或处理后）更新到上述的 Windows 和 Android C++ 文件中。

## 6. 编译与生效

修改 C++ 文件后：
*   **Android**: 运行 `flutter clean` 然后 `flutter run` 或 `flutter build apk`，Gradle 会自动调用 CMake 重新编译 `.so` 文件。
*   **Windows**: 运行 `flutter run -d windows` 或 `flutter build windows`，Visual Studio 会自动重新编译 `flutter_window.cpp`。
