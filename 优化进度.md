# VPN 优化与重构进度报告

本文档基于 `优化.md` 中的核心痛点，记录了我们在 Android 与 iOS 端，特别是 iOS `NetworkExtension` 和 `Mihomo` 核心层的重构进度。

## 一、Flutter 层（主 App）优化进度

### 1. 主进程与扩展进程的通信与解耦
- **状态：** ✅ **已完成**
- **措施：**
  - Flutter 端只负责配置生成、指令发送（通过 `MethodChannel`）以及状态显示。
  - 配置通过 `UserDefaults(suiteName: AppGroup)` 和共享文件目录双端读写。
  - 核心的 TUN 文件描述符（fd）操作、Mihomo 的生命周期已彻底隔离至 `PacketTunnelProvider`。

## 二、iOS 原生层（Runner / 主 App 原生）优化进度

### 1. TUN fd 与主进程隔离
- **状态：** ✅ **已完成**
- **措施：**
  - TUN 文件描述符的获取完全在 `PacketTunnelProvider` (NetworkExtension 扩展内部) 内部执行，符合 iOS 系统隔离规范，主程序不再染指任何 TUN 的创建与管理。

### 2. 第三方内核库升级与路径修复
- **状态：** ✅ **已完成**
- **措施：**
  - 静态库和头文件的引用路径从旧版的 `Runner/ThirdParty/mihomo` 迁移至 `Runner/ThirdParty` 与 `Runner/ThirdParty/include`。
  - Xcode 配置文件 (`project.pbxproj`) 中的 `HEADER_SEARCH_PATHS` 和 `LIBRARY_SEARCH_PATHS` 等编译选项已全面修复，清除了旧有库名带来的编译隐患。

## 三、NetworkExtension 拓展层（最核心的 FD 模式重构）

### 1. 取 fd 必须用 WireGuard utun 遍历法
- **状态：** ✅ **已完成**
- **措施：**
  - 在 `PacketTunnelProvider.swift` 中实现了 `getTunnelFd()` 方法，使用 `getsockopt` 函数和 0~1024 端口遍历法来精准捕获 iOS 底层的真实 `utun` 文件描述符，取代了已废弃且不稳定的 KVC 反射法。

### 2. 彻底抛弃官方读写方法 (packetFlow.readPackets)
- **状态：** ✅ **已完成**
- **措施：**
  - 完全删除了 `PacketFlowBridgeAdapter` 和相关的多线程读取包转发逻辑（`startReadPacketsLoop`）。
  - 内核通过读取配置文件中注入的 `file-descriptor: \(fd)`，直接接管底层网络读写，不再经过 Swift 层转发，极大地降低了 CPU 和内存开销，彻底规避了系统内存超 20MB 被杀的风险。

### 3. TUN 配置对齐 Android 与自动路由接管
- **状态：** ✅ **已完成**
- **措施：**
  - 在注入 `config.yaml` 的配置时，与 Android 端深度对齐，引入了 `gvisor` 网络栈。
  - 加入了以下核心参数，强制内核接管路由与 53 端口 DNS 请求，禁止多头管理带来的网络混乱：
    ```yaml
    tun:
      enable: true
      stack: gvisor
      file-descriptor: \(fd)
      auto-route: false
      auto-detect-interface: false
      auto-redirect: false
      mtu: 1500
      dns-hijack:
        - 0.0.0.0:53
        - "[::]:53"
    ```

### 4. 废弃过时 API 与网络切换自动重连机制
- **状态：** ✅ **已完成**
- **措施：**
  - 移除了由于内核更新而被废弃的 `MobileSetPacketFlowBridge`、`MobileMobileStartWithMemory`、`MobileResetNetwork` 等调用。
  - 现在使用 `MobileStart(home, configFileName)` 进行启动。
  - 针对 iOS 中**网络路径切换（如 Wi-Fi 切蜂窝）导致 VPN 断开**的痛点：当 `NWPathMonitor` 检测到网络路径变化时，会自动调用 `MobileForceUpdateConfig("config.yaml")` 强制重载配置并刷新网络状态，防止 VPN 断流。

---
**结论：** 针对 `优化.md` 中提出的所有 iOS 扩展限制、内存开销痛点及 fd 获取难题，已全部重构完成，并与最新的 Mihomo 核心接口全面适配。
